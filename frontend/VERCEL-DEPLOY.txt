  نشر الواجهة (React + Vite) على Vercel
═══════════════════════════════════════════════════════════════════

الـ API (Express) لا يعمل على Vercel كخادم دائم بنفس الطريقة؛ انشره على
Railway / Render / VPS أو أي VPS يشغّل Node طوال الوقت.
هذا الملف يخص الواجهة فقط — عنوان الـ API يبقى متغير بيئة.

───────────────────────────────────────────────────────────────────
  ■ كيف تترابط القطع (أنا ما أقدر أدخل حسابك — أنت تربطها بهالترتيب)
───────────────────────────────────────────────────────────────────

  المتصفح (أنت أو الموظفين)
        │
        │  يفتح رابط الواجهة
        ▼
  ┌─────────────┐     متغير واحد هنا: VITE_API_BASE_URL
  │   Vercel    │     = https://رابط-الـBACKEND-المنشور/api
  │  (React)    │
  └──────┬──────┘
        │  طلبات login وغيرها (HTTPS) ← axios
        ▼
  ┌─────────────┐     ملف .env على نفس السيرفر:
  │   Backend   │     قاعدة البيانات (MySQL / Postgres / … حسب backend/.env)
  │  (Express)  │     CLIENT_URL   ← رابط Vercel (CORS)
  └──────┬──────┘
        │  Sequelize / SQL
        ▼
  ┌─────────────┐
  │  قاعدة      │     مستضافة أو محلية — الواجهة HR من الـ API وليس من قاعدة البيانات مباشرة
  │  البيانات   │
  └─────────────┘

  • مزوّد الاستضافة لا يربط Vercel تلقائياً — أنت تربط:
      الواجهة → بـ Vercel + VITE_API_BASE_URL
      الـ API → بسيرفر + DATABASE_URL + CLIENT_URL
  • بدون Backend منشور على الإنترنت: الواجهة على Vercel ما تقدر تسجل دخول.

───────────────────────────────────────────────────────────────────
1) إعداد GitHub
   • ارفع المشروع إلى مستودع Git (أو اربط المجلد المحلي من Vercel Desktop).

2) مشروع جديد في Vercel
   • vercel.com → Add New → Project → استورد الريبو.
   • Root Directory: اتركه فارغاً (جذر الريبو). الإعداد الحالي يعتمد npm workspaces
     (package.json في الجذر + مجلد frontend) و build واحد من الجذر.
   • في Settings → General: إن وُجد Override لـ Framework أو Build Command يتعارض مع
     الريبو، عطّله أو اجعل Framework = Vite تلقائياً من الملف.

3) متغيرات البيئة (Vercel → Project → Settings → Environment Variables)
   أضف على الأقل:

   VITE_API_BASE_URL = https://عنوان-خادمك-الحقيقي.com/api
   (نفس الرابط الذي يشغّل الـ backend؛ مع /api في النهاية)
   • بدون هذا الرابط، الطلبات تذهب لنطاق Vercel نفسه → لن يعمل تسجيل الدخول (405/404).
     vercel.json لا يعيد كتابة /api/* إلى SPA لتفادي 405 الخادئة على POST.

   اختياري:
   VITE_APP_NAME
   VITE_DEFAULT_LANG=ar

   ── إكمال النشر على Vercel (لوحة المشروع) ──
   ① Settings → Environment Variables → Add New
   ② Key بالضبط:  VITE_API_BASE_URL
      Value:       https://عنوان-سيرفر-الـExpress-المنشور/api
      (مثال شكل: https://hr-api.onrender.com/api — استبدل بنطاقك)
   ③ Environments: Production و Preview (أو حسب حاجتك) → Save
   ④ Deployments → آخر نشر → ⋯ → Redeploy (حتى يُحقَن المتغير في build الـ Vite)
   ⚠ لا تضع DATABASE_URL أو DIRECT_URL أو JWT هنا — هذه لسيرفر الـ backend فقط (.env على
     Railway/Render/VPS)، وليس لمشروع الواجهة على Vercel.

4) Build
   • vercel.json في الجذر: framework = Vite، install = npm install، build = npm run build،
     output = frontend/dist (لا تختر «Other» أو Node في لوحة Vercel).
   • على Vercel أثناء npm run build: إن كان VITE_API_BASE_URL فارغاً أو نسبياً (/api)
     أو يشير لـ localhost، يفشل البناء برسالة من vite.config.mjs (أفضل من نشر واجهة
     معطوبة). لتجاوز غير مستحسن فقط: SKIP_VERCEL_API_CHECK=1 في Environment Variables.
   • Deploy — بعد النجاح تحصل على رابط مثل https://xxx.vercel.app

5) CORS على الـ backend (إجباري في الإنتاج)
   في ملف .env للخادم (ليس Vercel):

   NODE_ENV=production
   CLIENT_URL=https://xxx.vercel.app,https://xxx-git-main-xxx.vercel.app

   ضع كل الروابط التي يفتح منها المستخدم الواجهة (إنتاج + معاينة إن احتجت).
   بدونها المتصفح يمنع الطلبات (خطأ CORS).

   إن ظهر خطأ: Allow-Origin لا يجوز أن يكون '*' مع credentials — الـ backend يعيد أصلًا صريحًا من القائمة أعلاه.
   لروابط معاينة Vercel الطويلة (*.vercel.app) إما تضيفها كلها في CLIENT_URL مفصولة بفاصلة، أو على الـ backend فقط:
   ALLOW_VERCEL_PREVIEW_ORIGINS=1  (يسمح بأي https://*.vercel.app — أوسع من القائمة اليدوية).

6) الكوكيز / تسجيل الدخول
   الواجهة تستخدم withCredentials. في الإنتاج الـ backend يضبط SameSite للكوكي
   تلقائياً على none + Secure عندما تكون الواجهة على نطاق مختلف عن الـ API (مثل Vercel).
   إن كان كل شيء خلف نطاق واحد (Nginx): ضع في .env للخادم REFRESH_COOKIE_SAMESITE=strict
   راجع أيضاً CLIENT_URL لـ CORS.

7) اختبار محلي يشبه الإنتاج
   cd frontend
   npm run build && npm run preview
   مع VITE_API_BASE_URL يشير لسيرفر حقيقي أو ngrok.
